code/__HELPERS/matrices.dm

/proc/color_matrix_from_string Converts a hex colour string to a colour matrix.
/proc/color_matrix_recolor_rgb Creates a matrix to re-paint a sprite, replacing pure red, green, and blue with 3 different shades. Doesn't work with mixed tones of RGB or whites or greys -- must be pure. R/G/B 255 becomes the new colour, darker shades become correspondingly darker. The arg is a list of hex colours, for ex "list("#d4c218", "#b929f7", "#339933"". if you want variations of the same colour, color_matrix_recolor_red() is simpler.
/proc/color_matrix_recolor_red Creates a matrix to re-paint a sprite, replacing shades of red with corresponding shades of a new colour. In the base sprite, Hue must always be pure red. Saturation and Lightness can be anything. Arg is a hex string for a colour. Proc is by Lummox JR, www.byond.com/forum/post/2209545 color_matrix_recolor_rgb is more complex, but gives more precise control over the palette, at least if using 3 or fewer colours.
